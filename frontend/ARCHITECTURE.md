# 🏗️ Архитектура проекта Eli

## 📋 Обзор

**Eli** — интернет-магазин элитных елей с люксовым дизайном, внушающим доверие клиентам. Проект построен на современном стеке технологий с акцентом на производительность, типобезопасность и премиальный пользовательский опыт.

## 🛠️ Технологический стек

### Фронтенд

- **React 18** — UI библиотека с новейшими возможностями
- **TypeScript** — типобезопасность и лучший DX
- **Vite** — быстрая сборка и HMR
- **Tailwind CSS** — utility-first CSS фреймворк
- **React Router DOM** — клиентская маршрутизация
- **TanStack Query** — управление серверным состоянием
- **Axios** — HTTP клиент с интерцепторами
- **Framer Motion** — премиальные анимации

### Бэкенд

- **Node.js** — серверная платформа
- **Express** — HTTP фреймворк
- **TypeScript** — типобезопасность
- **Prisma** — ORM для работы с PostgreSQL
- **Zod** — валидация данных
- **JWT** — аутентификация

### Инфраструктура

- **PostgreSQL** — основная база данных
- **Redis** — кэширование и сессии

## 🏛️ Архитектурные принципы

### 1. Feature-Sliced Design (FSD)

```
frontend/src/
├── Pages/           # Страницы приложения
├── Shared/          # Переиспользуемые компоненты
├── Api/             # Слой работы с API
├── Config/          # Конфигурация
├── types/           # TypeScript типы
├── styles/          # Глобальные стили
└── assets/          # Статические ресурсы

backend/src/
├── routes/          # Маршруты API
├── controllers/     # Обработчики запросов
├── services/        # Бизнес-логика
├── models/          # Модели данных
├── middleware/       # Промежуточные обработчики
├── config/          # Конфигурация
└── types/           # Типы
```

### 2. Модульная архитектура

Каждая страница содержит:

- `ui/` — компоненты интерфейса
- `hooks/` — бизнес-логика
- `components/` — локальные компоненты

### 3. Разделение ответственности

- **Pages** — композиция компонентов
- **Shared** — переиспользуемая логика
- **Api** — работа с сервером
- **Controllers** — обработка HTTP
- **Services** — бизнес-логика

## 🌐 Работа с API

### HTTP клиент

```typescript
const apiClient = axios.create({
  baseURL: API_BASE_URL,
});

config.headers.Authorization = `Bearer ${token}`;

if (error.response?.status === 401) {
  handleUnauthorized();
}
```

### Работа с API через хуки

- Все обращения к API сервисам должны быть через хуки
- Компоненты не должны напрямую вызывать API сервисы
- Хуки инкапсулируют логику работы с данными

## 🧪 Качество кода

### Современный синтаксис

- **Стрелочные функции** — везде где возможно
- **Деструктуризация** — объектов и массивов
- **Template literals** — для строк
- **Optional chaining** — безопасный доступ к свойствам
- **Nullish coalescing** — обработка null/undefined
- **Async/await** — вместо промисов

### Именование

- **PascalCase** — компоненты, типы, интерфейсы
- **camelCase** — переменные, функции, хуки
- **UPPER_SNAKE_CASE** — константы
- **Префиксы хуков** — use* (useState, useAuth)
- **Булевы переменные** — is*, has*, should*
- **Обработчики событий** — handle*

### Размер файлов

- Максимум 200 строк на файл (где возможно)
- При превышении — разбивать на компоненты/хуки

### DRY принцип

- Не дублировать код более 2 раз
- Извлекать в функции/компоненты/хуки

### Комментарии

- Код должен быть самодокументируемым
- Комментарии только для сложной бизнес-логики
- Удалять закомментированный код

### Безопасность

- Никогда не хранить токены в localStorage (только sessionStorage)
- Валидировать все пользовательские данные
- HTTPS для всех API запросов
- Санитизировать HTML контент

## 📈 Масштабируемость

### Добавление новых фич

1. Создать папку в `Pages/`
2. Добавить API сервисы в `Api/`
3. Добавить роуты на бэкенде
4. Определить типы в `types/`
5. Создать маршрут в `App.tsx`

### Переиспользование

- Компоненты в `shared/components/`
- Хуки в `shared/hooks/`
- Утилиты в `utils/`
